# 浏览器智能 Agent (Browser Agent) 简要需求文档 (PRD)

## 1. 项目概述
本项旨在开发一个基于 LangGraph 的自动化浏览器 Agent。该 Agent 能够像人类一样“观察”网页内容、进行逻辑“思考”、并执行相应的浏览器“操作”，以完成复杂的跨页面任务。

## 2. 核心架构 (基于 LangGraph)
系统采用有状态的循环图结构：
- **Nodes (节点)**: 观察 (Observe)、思考 (Thinking)、执行 (Actions)。
- **Edges (边)**: 节点间的状态流转，支持根据思考结果决定是继续行动还是结束任务。

## 3. 功能模块详细说明

### 3.1 观察模块 (Observe) - 输入端
该模块负责为 LLM 提供决策所需的上下文。
* **元素定位 (LLM + RAG)**: 
    * 针对复杂的 DOM 树，利用 RAG 检索技术提取相关的 HTML 代码片段。
    * 结合 LLM 语义化识别目标按钮、输入框等元素，提高在动态页面下的鲁棒性。
* **视觉感知 (看页面有什么)**: 
    * 获取当前页面的截图或 Accessibility Tree（无障碍树），理解页面布局和状态。
* **记忆检索 (找之前保存的信息)**: 
    * 从短期记忆或向量数据库中提取此前步骤获取的关键数据。
* **效果确认**: 
    * 在每次 Action 执行后，对比执行前后的页面差异，确认操作是否生效（如：点击后是否跳转，弹窗是否关闭）。

### 3.2 思考模块 (Thinking) - 核心逻辑
* **任务拆解**: 将用户的大目标拆解为可执行的微步骤。
* **状态维护**: 维护当前的 Task Progress，判断是否已达成目标。
* **决策引擎**: 根据“观察”到的信息，决定下一步是调用具体的 API，还是需要重新寻找元素。

### 3.3 执行模块 (Actions) - 输出端
* **浏览器交互 (Browser Actions)**: 
    * 封装 Playwright 或 Puppeteer 操作。
    * 支持：Click (点击), Type (输入), Scroll (滚动), Hover (悬停), Wait (等待) 等。
* **信息提取与保存 (OCR)**: 
    * 对于非文本形式的内容（如图片中的文字、复杂图表），通过 OCR 技术进行结构化提取。
    * 将提取的信息存入状态机或外部数据库供后续步骤使用。

## 4. 业务流程 (Workflow)
1. **Start**: 接收用户指令（例如：“去某网站订一张明天的机票”）。
2. **Thinking**: 分析当前处于什么页面，下一步需要做什么。
3. **Observe**: 扫描页面，利用 RAG 找到对应的订票入口或输入框。
4. **Actions**: 执行点击或输入动作。
5. **Verify**: 观察操作后的反馈（效果确认）。
6. **Loop**: 重复上述过程，直到任务完成。
7. **End**: 返回最终结果给用户。

## 5. 技术关键点
* **LangGraph 状态管理**: 定义全局状态对象 `AgentState`，记录任务历史、保存的 OCR 信息和当前页面截图。
* **上下文压缩**: 由于浏览器 DOM 往往非常庞大，必须通过 RAG 或特定算法清洗 HTML，只保留必要信息以节省 Token。
* **异常处理**: 当 LLM 无法在页面找到元素时，需具备“重试”或“报错”的逻辑分支。

## 6. 未来扩展
* 多标签页切换与并行处理能力。
* 集成更多外部工具（如：验证码绕过、文件下载与处理）。
